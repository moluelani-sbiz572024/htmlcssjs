FRONTEND LAYER

HTMLとCSS、JavaScriptで構成される利用者と最も隣接するシステム境界であり、システムの「窓口」ともなるレイヤーです。 所謂「GUI」（単にUIと呼ぶ場合もある）であり、利用者にとってシステムの利便性が最も現れる場所でもあります。よって、 利便性の良し悪しで「システム評価」に直結しやすく、操作性の確保は非常に重要な要件となっています。

BACKEND LAYER

システムのデータ領域でありシステムが利用する、または利用者がGUI経由で入力したデータを格納し管理する場所でもあります。 所謂「インテグレーション」レイヤーと呼ばれる場所で主にデータベースシステムで構成されます。データベースとしては、 RDBMS(Relational Database Management System)が配置されることが多いですが、最近ではNoSQLと呼ばれるRDBMSとは異なる データ格納領域を配置するケースもあります。代表的なものとして、MongoDBがあります。RDBMSとしては、PostgreSQLやMySQLが よく利用されます。

PROCESSING LAYER

FRONTENDから入力されたデータは、リクエスト情報としてサーバー領域に送信されます。リクエスト情報を受け取りシステムとして 必要な処理を実行して結果をレスポンス情報として加工した上でリクエスト元へ返却送信します。この一連の流れをWebシステムでは HTTPベースでリクエスト・レスポンス処理と呼びます。この流れの中には、他のシステム呼び出しやサブシステムなどの利用のほか、 BACKENDへの連携によってデータ領域からデータ取得したりデータ領域への保管なども行います。システム仕様の中心的存在でもあります。
